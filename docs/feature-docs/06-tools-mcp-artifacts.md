# Tools, MCP, Artifacts & Session — Deep Dive

EdgeCrab implements a robust, extensible tool system, dynamic MCP integration, and a session/artifact management layer with checkpoint/rollback. This doc provides a code-backed, architectural deep dive with citations and TODOs for further parity.

---

## 1. Tool Registry Architecture

- **Trait-based, compile-time registration:**
	- All tools implement the `ToolHandler` trait ([registry.rs](../../crates/edgecrab-tools/src/registry.rs)).
	- Registration uses the `inventory` crate for zero-cost, linker-guaranteed inclusion (no runtime registration bugs).
	- Dynamic tools (plugins, MCP) are supported via runtime registration (`register_dynamic`).
- **Dispatch:**
	- O(1) lookup by name, toolset-based filtering, fuzzy matching for typos (Levenshtein, `fuzzy_match`).
	- Parallel safety: `parallel_safe()` trait method allows safe concurrent execution for certain tools.
	- Tool gating: `is_available()` (startup) and `check_fn()` (per-request) for runtime policy enforcement.
- **Context:**
	- `ToolContext` struct provides per-call context: cwd, session/task ID, config, process table, provider, registry, todo store, etc.
	- All tool calls are mediated by the registry, enforcing security and policy.

## 2. Core Toolsets & Extensibility

- **Core toolsets:**
	- File: `read_file`, `write_file`, `patch`, `search_files`, `pdf_to_markdown`
	- Terminal: `terminal`, `run_process`, `kill_process`, etc.
	- Web: `web_search`, `web_extract`, `web_crawl`
	- Browser: `browser_*` (automation, vision)
	- Skills: `skills_list`, `skill_view`, `skill_manage`, `skills_hub`, `skills_guard`, `skills_sync`
	- Session: `session_search`, `checkpoint`, `manage_todo_list`
	- Memory: `memory_read`, `memory_write`
	- Delegation: `delegate_task`, `mixture_of_agents`
	- Code execution: `execute_code` (sandboxed, RPC tool access)
	- MCP: `mcp_list_tools`, `mcp_call_tool`, `mcp_list_resources`, etc.
	- Media: `vision_analyze`, `text_to_speech`, `transcribe_audio`, `generate_image`
	- Automation: `manage_cron_jobs`
- **Extensibility:**
	- New tools: implement `ToolHandler`, register in `tools/mod.rs`, add to toolset if needed.
	- Skills: YAML-frontmatter skills in `~/.edgecrab/skills/`, managed via tools and skills hub.
	- Plugins: dynamic tool registration at runtime.

## 3. MCP Integration (Model Context Protocol)

- **Client:**
	- Implements both stdio and HTTP JSON-RPC 2.0 clients ([mcp_client.rs](../../crates/edgecrab-tools/src/tools/mcp_client.rs)).
	- Connection pooling via DashMap, token management (`read_mcp_token`, `write_mcp_token`).
	- Tool calls: `mcp_list_tools`, `mcp_call_tool`, etc. — discover and invoke remote tools.
	- Operator workflows now include shared MCP doctor diagnostics, actionable MCP auth guidance, interactive OAuth login, refresh-token caching, and quote-aware TUI command parsing for path-bearing install flows.
- **TUI / CLI control plane:**
	- `crates/edgecrab-cli/src/app.rs` exposes `/mcp` as a searchable MCP browser with install, view, test, diagnose, and remove actions.
	- `/mcp search` opens a dedicated remote browser for official upstream sources plus the official MCP Registry, with per-source provenance and install/view actions.
	- `crates/edgecrab-cli/src/main.rs` exposes the same control plane via `edgecrab mcp ...`, including `edgecrab mcp doctor`, `edgecrab mcp auth`, `edgecrab mcp login`, and grouped multi-source `edgecrab mcp search`.
- **Server:**
	- ACP adapter in `edgecrab-acp` crate (not detailed here).
	- Allows VS Code and other agents to connect and use EdgeCrab as a tool server.
- **Security:**
	- OAuth token storage in `~/.edgecrab/mcp-tokens/` (0600 perms), per-server isolation, including refresh-token caching for refresh-token grant flows.
	- HTTP MCP configs may use env-backed bearer tokens and header values, which are expanded at load time for OAuth-style access-token flows.
	- All tool calls pass through registry security checks.

## 4. Session, Artifacts, Checkpoints

- **Session state:**
	- SQLite WAL + FTS5 session store (`edgecrab-state` crate), full-text search via `session_search` tool.
	- Per-session todo/task state (`TodoStore`), survives context compression.
- **Artifacts:**
	- Generated by tools (e.g., `generate_image`, `transcribe_audio`), saved to `~/.edgecrab/generated/` or session-specific locations.
	- Naming: `MEDIA:/absolute/path` for gateway delivery; sequential/human-readable names for files.
- **Checkpoints & rollback:**
	- `checkpoint` tool: filesystem snapshot, rollback support.
	- Mirrors hermes-agent's checkpoint/restore pattern.
- **Trace capture:**
	- (TODO) EdgeCode has richer trace capture/redaction; EdgeCrab currently focuses on session/task state and SQLite logs.

## 5. Security & Policy

- **Tool gating:**
	- Per-tool and per-toolset enable/disable, runtime gating, parallel safety.
- **Skill scanning:**
	- All external skills scanned for 23+ threat patterns (exfiltration, injection, destructive, persistence, obfuscation) before install ([skills_guard.rs](../../crates/edgecrab-tools/src/tools/skills_guard.rs)).
- **Path safety:**
	- All file I/O validated via `edgecrab-security` (not detailed here).
- **SSRF & command scan:**
	- All web/terminal tools pass SSRF and command-injection checks.
- **Audit logs:**
	- Skills hub appends every install/uninstall to a JSON audit log for traceability.

## 6. TODOs & Parity Gaps

- **Trace capture:**
	- EdgeCode implements full LLM trace capture with redaction; EdgeCrab could add this for parity.
- **Artifact naming:**
	- EdgeCode uses sequential, LLM-optimized artifact naming; EdgeCrab could adopt similar conventions.
- **Redaction policy:**
	- EdgeCode redacts secrets/tokens in traces/artifacts; EdgeCrab could extend redaction beyond tool output.
- **Session directory structure:**
	- EdgeCode has per-session directories for logs/artifacts/traces; EdgeCrab could further modularize session storage.
